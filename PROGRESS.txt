PHASE 5.5 — Accuracy Overhaul     [DONE] 2026-03-19
=========================================
Анализ 21 старой ставки выявил критические проблемы:
  - 16/21 ставок на рынках с market_prob < 1% (мусорные tail events)
  - Дубли: один рынок до 4 раз за день
  - Claude переоценивал маловероятные события (0.05% → 75%)
  - P&L расчёт сломан для NO ставок
  - Ноль отслеженных результатов

--- TIER 1: Критические баг-фиксы ---
[x] Фильтр экстремальных цен: skip если yes_price < 5% или > 95%
[x] Дедупликация: has_recent_bet() — один condition_id раз в 24ч
[x] Фикс market_prob: всегда хранит yes_price, P&L для NO через 1-market_prob
[x] Валидация condition_id: пропуск если < 10 символов

--- TIER 2: Улучшение точности Claude ---
[x] Новый промпт: английский, якорение к рыночной цене, лимиты отклонения
[x] Модель Sonnet вместо Haiku (конфигурируется через CLAUDE_MODEL)
[x] Hard cap: отклонение от рынка не больше 30%

--- TIER 3: Управление рисками ---
[x] Дневной лимит ставок (DAILY_BET_LIMIT=$25)
[x] Макс открытых позиций (MAX_OPEN_BETS=10)

--- TIER 4: Отслеживание результатов ---

[x] Cooldown 1ч для неразрешённых рынков (не долбить API)

PHASE 5.6 — Throughput Fix        [DONE] 2026-03-19
=========================================
Проблема: 3 ставки за 30 циклов — бот пропускал почти все рынки.
Причины: GNews 429, кэш пустых ответов на 30мин, "кэш, пропуск" блокировал Claude.

[x] Пустые ответы GNews кэшируются на 2 мин вместо 30 (EMPTY_CACHE_TTL)
[x] Кэшированные новости НЕ блокируют Claude — анализ с пометкой "(кэш)"
[x] Крипто рынки анализируются без новостей (CoinGecko + Fear & Greed достаточно)
[x] Задержка 1с между GNews запросами (защита от 429)
[x] Параллельный сбор данных (новости + цена + крипто + F&G одновременно)
[x] Промпт: confidence="low" только при полном отсутствии данных
[x] CryptoPanic API — крипто-новости с sentiment (bullish/bearish) для крипто рынков
    Приоритет: CryptoPanic → Tavily → GNews (fallback цепочка)
